Autentikasi API

Cara mengautentikasi permintaan API Anda

Semua permintaan API memerlukan autentikasi menggunakan API Key. Sertakan API Key Anda di header setiap permintaan.

Header Authentication 
Authorization: Bearer YOUR_API_KEY
Content-Type: application/json
X-Client-Version: 3.0.0

Keamanan API Key

Jangan pernah membagikan API Key Anda atau menyimpannya di kode client-side (frontend). Gunakan server-side untuk semua permintaan API.

API Endpoints

Daftar endpoint API yang tersedia

Method Endpoint Deskripsi
GET /api/v3/games Daftar semua game tersedia
GET /api/v3/games/:slug/products Daftar produk per game
POST /api/v3/orders Buat order baru
GET /api/v3/orders/:id Cek status order
GET /api/v3/balance Cek saldo akun
POST /api/v3/premium/subscribe Langganan aplikasi premium
GET /api/v3/transactions Riwayat transaksi
PUT /api/v3/webhooks Update webhook URL

Response Format

Format response standar dari semua endpoint API

Success Response (200) 
{
  "status": "success",
  "code": 200,
  "message": "Order berhasil dibuat",
  "data": {
    "orderId": "TRX-20260429-XXXXXX",
    "game": "mobile-legends",
    "userId": "123456789",
    "item": "86 Diamonds",
    "price": 19500,
    "status": "pending",
    "paymentUrl": "https://pay.topupku.id/...",
    "createdAt": "2026-04-29T10:30:00Z"
  }
}
Error Response (400) 
{
  "status": "error",
  "code": 400,
  "message": "User ID tidak ditemukan",
  "errors": [
    {
      "field": "userId",
      "message": "Format User ID tidak valid"
    }
  ]
}
